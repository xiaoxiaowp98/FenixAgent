# Plan 09：RCS 是配置的单一权威来源，运行时注入

## Context

当前 RCS 管理 Provider/Model/AgentConfig/Skill/McpServer 等配置，但 Instance spawn 时注入不完整（只有 default_agent 和 KB MCP 端点）。需要让 RCS 成为所有配置的**单一权威来源**，在 spawn 时完整注入。

## 现状分析

### 当前注入逻辑

`src/services/instance.ts` 的 `spawnInstanceFromEnvironment()` 中：

```typescript
// 当前只注入了：
// 1. default_agent（通过 agentName）
// 2. KB MCP 端点
// 缺失：
// - Provider/Model 配置
// - Skill 内容
// - MCP 服务器配置
// - Permission 规则
// - 完整的 AgentConfig 配置
```

### workspace 配置文件

Agent workspace 里的 `.opencode/opencode.json` 当前部分由用户编辑、部分由 RCS 生成。目标：变为 RCS 自动生成的只读文件。

### 涉及文件

| 文件 | 改动 |
|------|------|
| `src/services/instance.ts` | 扩展 `spawnInstanceFromEnvironment()` 的注入逻辑 |
| `src/services/config-pg.ts` | 新增批量读取接口 |
| `src/services/workspace-config.ts` | 新增 workspace 配置生成器（可选） |

## 具体实施步骤

### Step 1：新增批量配置读取接口

```typescript
// src/services/config-pg.ts
interface AgentFullConfig {
  agentConfig: AgentConfig;
  providers: Provider[];
  skills: Skill[];
  mcpServers: McpServer[];
  // knowledgeBindings: KnowledgeBinding[]; // 如果有
}

async function getAgentFullConfig(userId: string, agentConfigId: string): Promise<AgentFullConfig>
```

### Step 2：扩展 spawn 注入逻辑

```typescript
// src/services/instance.ts - spawnInstanceFromEnvironment()
// 完整注入：
// 1. AgentConfig 所有配置（model, prompt, steps, permission 等）
// 2. Provider + Model 配置
// 3. Skill 内容（从文件系统读取）
// 4. MCP 服务器配置
// 5. Knowledge MCP 端点
// 6. Permission 规则
```

### Step 3：生成 workspace 配置文件

```typescript
// 生成 .opencode/opencode.json
// 包含完整配置，标记为 RCS 自动生成（只读）
// 添加文件头注释说明不要手动编辑
```

### Step 4：防止用户直接编辑

- workspace 配置文件添加注释标记
- 文件头写入 `// AUTO-GENERATED BY RCS — DO NOT EDIT`
- spawn 时覆盖已有的手动配置

## 验证方式

```bash
# 单元测试
bun test src/__tests__/

# 集成验证
bun run dev
# 1. 配置完整的 Agent（Provider + Model + Skill + MCP）
# 2. 创建 Environment 并 spawn Instance
# 3. 检查 workspace 的 .opencode/opencode.json 内容
# 4. 验证所有配置维度都已注入
```

## 风险点

1. **配置覆盖**：用户手动编辑的 workspace 配置会被覆盖，需要明确的迁移/通知策略
2. **配置量过大**：大量 Skill/MCP 配置可能超出 workspace 配置文件限制
3. **实时同步**：spawn 后修改配置如何更新已运行的 Instance？需要重载机制
