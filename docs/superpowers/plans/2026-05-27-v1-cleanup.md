# V1 代码清理实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 一次性删除所有 v1 控制面板界面和后端 API 路由，保留 v2 使用的代码。

**Architecture:** 删除后端 `src/routes/v1/` 中 3 个路由文件（保留并搬迁 session-ingress 到 v2 目录），删除前端 `web/src/routes/_app/` 整个目录及关联页面和组件。删除操作按后端→SDK→前端顺序执行，最后用 precheck + tests + build 验证。

**Tech Stack:** Elysia (后端路由)、TanStack Router (前端路由)、Drizzle/PostgreSQL (数据)、React 19 (前端)

`src/routes/v1/session-ingress.ts` 实际挂载的是 `/v2/session_ingress`，文件位置应与其路径一致。必须先搬迁再删 v1 目录，避免误删。

**Files:**
- Move: `src/routes/v1/session-ingress.ts` → `src/routes/v2/session-ingress.ts`
- Modify: `src/index.ts:21`

- [ ] **Step 1: 移动文件**

```bash
mv src/routes/v1/session-ingress.ts src/routes/v2/session-ingress.ts
```

- [ ] **Step 2: 更新 src/index.ts 中的 import 路径**

将第 21 行：
```typescript
import v1SessionIngress from "./routes/v1/session-ingress";
```
改为：
```typescript
import sessionIngress from "./routes/v2/session-ingress";
```

同时更新挂载处（约第 159 行）：
```typescript
.use(v1SessionIngress)
```
改为：
```typescript
.use(sessionIngress)
```

- [ ] **Step 3: 验证移动后 tsc 通过**

Run: `bunx tsc --noEmit`
Expected: 无报错

- [ ] **Step 4: Commit**

```bash

git rm src/routes/v1/session-ingress.ts

```

precheck 是代码质量第一标准：biome format → import 排序 → tsc → biome check。如果有遗漏的引用，tsc 会直接报错。

- [ ] **Step 1: 运行 precheck**

Run: `bun run precheck`

Expected: 全部通过。如果 tsc 报错，说明有文件仍然引用了被删除的模块，需要逐个修复。

常见修复模式：
- 如果有文件 import 了 `StatusBadge` from `Navbar`，改用 `web/components/config/StatusBadge.tsx` 中的替代
- 如果有文件 import 了被删除的页面/组件，直接删除该 import（该文件可能也是 v1 遗留）
