# v2 智能体列表 UI 优化设计

> 日期：2026-05-27
> 状态：已确认

## 概述

优化 v2 Agent 面板侧边栏的智能体列表交互模型，解决当前列表混乱的问题。核心改动：
1. 点击智能体名字直接进入默认实例（而非展开/折叠）
2. 左侧 chevron 控制展开/折叠，展开后显示实例列表
3. 智能体行和实例行增加重启/停止按钮
4. AgentConfigDialog 保存后提示用户是否重启实例

## 当前行为

- 点击智能体整行 header → 展开/折叠实例列表
- 展开后顶部显示"新建实例"按钮 + 实例列表
- 实例无操作按钮，只能点击进入
- AgentConfigDialog 保存后仅显示 toast，直接关闭

## 改造设计

### 智能体行布局

```
折叠状态：
┌──────────────────────────────────────┐
│ [▸] 🤖 MyAgent           [↻] [⚙]   │
└──────────────────────────────────────┘

展开状态：
┌──────────────────────────────────────┐
│ [▾] 🤖 MyAgent           [↻] [⚙]   │
├──────────────────────────────────────┤
│  ● 实例 #1          [↻] [■]  (hover)│
│  ● 实例 #2          [↻] [■]  (hover)│
│  [+ 新实例]                          │
└──────────────────────────────────────┘
```

**交互区域**：

| 区域 | 点击行为 |
|------|----------|
| `[▸]/[▾]` 左侧 chevron | 展开/折叠实例列表 |
| 智能体名字区域 | 进入默认实例（聊天页面） |
| `[↻]` 重启按钮 | 重启实例（见下方重启逻辑） |
| `[⚙]` 设置按钮 | 打开 AgentConfigDialog |

### 默认实例

点击智能体名字时进入的实例。定义：该智能体关联 environment 下 `instance_number` 最小的**运行中**实例。如果没有运行中实例，则先 spawn 一个再进入（沿用现有 `handleEnterAgent` 逻辑）。

### 实例行

- 点击实例行 → 进入该实例
- `[↻]` 和 `[■]` 按钮 hover 时显示（`group-hover`）
- `[↻]` 重启该实例（直接执行，无确认弹窗）
- `[■]` 停止该实例（直接执行，无确认弹窗）
- "新实例"按钮位于列表**底部**（最后一个位置）

### 重启逻辑

前端组合调用（无新后端接口）：
1. `instanceApi.delete({ id })` 停止实例
2. `instanceApi.spawn({ environmentId })` 创建新实例
3. 如果用户正在该实例的聊天页面，重启后自动重新进入新实例

**智能体行 [↻] 按钮**：
- 1 个运行中实例 → 直接重启
- 多个运行中实例 → 弹出 AlertDialog，列出所有运行中实例，checkbox 默认全选，用户确认后批量重启

**实例行 [↻] 按钮** → 直接重启该实例，无确认弹窗

### 停止逻辑

**实例行 [■] 按钮**：
- 直接停止，调用 `instanceApi.delete`
- 如果用户正在该实例的聊天页面，停止后清空当前 chat 状态

### AgentConfigDialog 保存后提示

保存成功后不再直接关闭弹窗，改为：

1. 配置保存成功 → toast "配置已保存"
2. 弹出确认对话框："配置已保存，需要重启智能体实例才能生效。是否立即重启？"
3. 重启逻辑复用同一套（单实例直接重启，多实例 AlertDialog 勾选）
4. 用户选择"稍后" → 关闭弹窗，不重启
5. 用户选择"重启" → 执行重启，完成后关闭弹窗并刷新列表
